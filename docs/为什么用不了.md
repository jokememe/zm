# 到底为什么用不了（说人话）

## 先说结论

我这边**实测**了你的中转：

```text
http://38.244.63.197:15511/v1/models
→ HTTP 401（未提供令牌）
→ 响应头有 X-New-Api-Version（New API / One API）
```

说明：**中转服务器是活的、公网能连上**。  
问题不在「模型名写错」本身，而在：**浏览器里的网站 → 中转** 这一段路径断了。

---

## 三条断点（按概率）

### 1）你在 HTTPS 网站上直连 HTTP 中转（必挂）

| 页面 | 中转 |
|------|------|
| `https://xxx.vercel.app` 或 `https://xxx.github.io` | `http://38.244.63.197:15511` |

浏览器规则：**HTTPS 页面禁止请求 HTTP 接口**（混合内容）。  
表现：`Failed to fetch`。

这和写多少代码无关，浏览器直接拦。

### 2）走了不存在的代理地址（你看到的 404）

你报过：

```text
/api/llm/v1/chat/completions HTTP 404
{"error":{"code":"404","message":"The page could not be found"}}
```

这是 **Vercel 说「没有这个页面/函数」**，  
**请求根本没到 38.244 那台机器**。

原因：旧方案用 `/api/llm/...` 动态路由，在 Vercel 上经常注册失败。  
现已改成固定：

- `POST /api/chat`
- `GET /api/models`
- `GET /api/health` ← 用来自测代理在不在

### 3）部署平台不对 / 没重新部署

| 平台 | 有没有 /api 代理 |
|------|------------------|
| **GitHub Pages** | **没有**，永远不能靠代理修混合内容 |
| **Vercel** | **有**，但要部署**最新代码**且 Functions 里看得到 `chat` / `models` |

---

## 你怎么 30 秒判断卡在哪

打开你的站点（必须是 `https://xxx.vercel.app`，不是 github.io）：

1. 浏览器打开：`https://你的站/api/health`  
   - **有** `{"ok":true,...}` → 代理函数已部署  
   - **404 HTML/JSON page not found** → 还在旧部署 / 不是 Vercel / 函数没起来  

2. 打开：`https://你的站/api/models`  
   - **401 未提供令牌**（中文 new_api 报错）→ 代理通了，只差 Key  
   - **404 page could not be found** → 还是函数没挂上  
   - **502 proxy_failed** → 函数在，但 Vercel 连不上 38.244（防火墙/IP）  

3. 密匣填：

```text
Base URL: http://38.244.63.197:15511/v1
API Key:  你的真 key
模型:     中转里有的模型名
```

在 HTTPS 下会自动改走 `/api/chat`，**不要**再填 `/api/llm/...`。

## 和「胡扯」的区别

| 说法 | 真假 |
|------|------|
| 中转挂了 | **假**（实测 401，是活的） |
| GitHub Pages 直连 HTTP 能用 | **假**（浏览器禁止） |
| 只要改模型名就能在 Pages 用 | **假** |
| Vercel + `/api/chat` 代理可以 | **真**（前提：部署成功） |
| 旧 `/api/llm/...` 404 | **真**（路由没了，已改） |

---

## 最稳用法（别绕）

1. 仓库最新代码推到 GitHub  
2. **只**用 Vercel 部署这个仓库  
3. 打开 `/api/health` 确认 ok  
4. 密匣填 `http://38.244.63.197:15511/v1` + Key + 模型  
5. 测试  

本机：`npm run dev`，同样填上面地址（走 Vite 代理到同一中转）。
